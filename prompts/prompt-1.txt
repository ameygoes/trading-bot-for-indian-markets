STOCK CLASSIFICATION BY MARKET CAP:

  > 20,000 Cr  →  SWING TRADE    hold 5–30 days   position 4% portfolio
  > 5,000 Cr   →  SWING LIGHT    hold 3–10 days   position 2.5% portfolio
  < 5,000 Cr   →  DAY TRADE      intraday only    position 1.5%, strict 1% SL

SIGNAL REQUIREMENTS:
  Swing     : 3+ indicator alignment + volume confirm + research conviction >= MEDIUM
  Day trade : fast signals on 1min/5min chart, strict 1% stop loss, no overnight hold

ENTRY / EXIT RULES:
  Entry     : ATR-based zone
  Stop loss : 1.5x ATR below entry (swing) | 1% hard stop (day trade)
  Target    : minimum 1:2 risk-reward ratio
  Sizing    : 2% portfolio risk per trade (fixed fractional, Kelly-informed)
  Trailing  : activate trailing stop when Target 1 is hit, move SL to entry

### Architecture: provider-agnostic notification layer

Build a clean abstract base so any provider can be swapped or added with
zero changes to the rest of the system. This also makes the entire approval
flow portable to OpenClaw or any other agentic workflow orchestrator later.

File: `notifications/base.py`

  class NotificationProvider(ABC):
    @abstractmethod
    async def send_message(self, text: str, parse_mode: str) -> str:
      """Send a message. Returns message_id."""

    @abstractmethod
    async def send_document(self, content: str, filename: str) -> str:
      """Send a file/document."""

    @abstractmethod
    async def poll_reply(self, message_id: str, timeout_seconds: int) -> str | None:
      """Poll for a user reply to a specific message. Returns reply text or None."""

    @abstractmethod
    async def send_alert(self, text: str) -> None:
      """Fire-and-forget urgent alert."""
